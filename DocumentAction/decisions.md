# Технические решения проекта Stroiuslugi

Документ фиксирует ключевые технические решения, принятые при разработке проекта, на основе анализа кода.

## 1. Выбор технологий

### Frontend Framework
**Решение**: Bootstrap 5.3.2

**Обоснование**:
- Использование Bootstrap 5 для создания адаптивного дизайна
- Подключение через CDN: `https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css`
- Использование grid system Bootstrap для раскладки компонентов
- Компоненты Bootstrap: navbar, modal, carousel, accordion, dropdown

**Код подтверждающий**:
```html
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet">
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"></script>
```

### Иконки
**Решение**: Bootstrap Icons 1.11.1

**Обоснование**:
- Единый стиль иконок, совместимый с Bootstrap
- Подключение через CDN
- Использование иконок в навигации, карточках, формах

**Код подтверждающий**:
```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.1/font/bootstrap-icons.css">
```

### JavaScript
**Решение**: Vanilla JavaScript (без фреймворков)

**Обоснование**:
- Отсутствие зависимостей от внешних библиотек
- Нативный JavaScript для простых задач
- Использование Bootstrap Modal API для модальных окон
- HTML5 валидация форм без дополнительных библиотек

**Модульная структура**:
- **main.js** - общая логика для всех страниц
- **services-gallery.js** - специализированный модуль для галерей услуг
- Разделение ответственности для лучшей поддерживаемости

## 2. Архитектурные решения

### Структура проекта
**Решение**: Многостраничная структура со статическими HTML файлами

**Обоснование**:
- Каждая страница - отдельный HTML файл
- Отсутствие серверной логики
- Готовность к размещению на GitHub Pages или статическом хостинге
- Простота развертывания и поддержки

**Структура**:
```
/
├── index.html (главная)
├── about.html, contact.html, portfolio.html (основные страницы)
└── services/ (10 страниц услуг)
```

### Организация стилей
**Решение**: Один файл кастомных стилей `css/style.css`

**Обоснование**:
- Все кастомные стили в одном файле для простоты
- Использование CSS-переменных для цветовой схемы
- Дополнение Bootstrap стилями, не переопределение
- Легкость поддержки и изменения

**Код подтверждающий**:
```css
:root {
    --primary-color: #ff6b35;
    --secondary-color: #004e89;
    --dark-color: #1a1a1a;
    --light-color: #f8f9fa;
    --text-color: #333;
    --border-color: #e0e0e0;
}
```

### Организация скриптов
**Решение**: Один файл `js/main.js` с общей логикой

**Обоснование**:
- Централизованная логика для всех страниц
- Функции для переиспользования (openOrderModal, initLightbox)
- Простота поддержки
- Отсутствие дублирования кода

### Организация изображений
**Решение**: Папки по категориям услуг в `img/`

**Обоснование**:
- Логическая группировка изображений
- Легкость навигации и поиска
- Масштабируемость при добавлении новых изображений
- Использование формата WebP для оптимизации

**Структура**:
```
img/
├── painting/
├── construction/
├── facade/
└── ...
```

## 3. Дизайн-решения

### Цветовая схема
**Решение**: Кастомная цветовая палитра через CSS-переменные

**Цвета**:
- Primary: `#ff6b35` (оранжевый)
- Secondary: `#004e89` (синий)
- Dark: `#1a1a1a` (темно-серый)
- Light: `#f8f9fa` (светло-серый)

**Обоснование**:
- Яркий primary цвет для привлечения внимания
- Профессиональная цветовая схема
- Контрастность для читаемости

### Типографика
**Решение**: Системный шрифт 'Segoe UI' с fallback

**Код подтверждающий**:
```css
body {
    font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
}
```

### Градиенты
**Решение**: Использование градиентов для hero и CTA секций

**Код подтверждающий**:
```css
background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
```

### Тени и эффекты
**Решение**: Использование box-shadow для глубины

**Применение**:
- Карточки услуг с тенями
- Hover-эффекты с увеличением тени
- Модальные окна с тенями

## 4. UX/UI решения

### Фиксированная навигация
**Решение**: Фиксированная navbar в верхней части страницы

**Обоснование**:
- Всегда доступная навигация
- Улучшение UX при прокрутке
- Стандартная практика для веб-сайтов

**Код подтверждающий**:
```html
<nav class="navbar navbar-expand-lg navbar-light bg-white fixed-top">
```

### Модальные окна для форм
**Решение**: Использование Bootstrap Modal для форм заявок

**Обоснование**:
- Не прерывает навигацию пользователя
- Фокус на форме
- Легкость реализации через Bootstrap
- Центрирование на экране

### Валидация форм
**Решение**: HTML5 валидация с визуальной обратной связью

**Обоснование**:
- Нативная поддержка браузерами
- Не требует дополнительных библиотек
- Визуальная обратная связь через Bootstrap классы
- Паттерн валидации для телефона

**Код подтверждающий**:
```html
<input type="tel" class="form-control" id="clientPhone" name="phone" required pattern="[0-9+\s()-]+">
<div class="invalid-feedback">Пожалуйста, введите корректный номер телефона.</div>
```

### Lightbox для галереи
**Решение**: Динамическое создание Bootstrap Modal для просмотра изображений

**Обоснование**:
- Переиспользование существующих компонентов Bootstrap
- Простота реализации
- Адаптивность из коробки

## 5. SEO решения

### Мета-теги
**Решение**: Уникальные meta-теги для каждой страницы

**Реализация**:
- Уникальный `<title>` для каждой страницы
- Уникальный `<meta name="description">`
- `<meta name="keywords">` для каждой страницы
- Open Graph мета-теги для социальных сетей

**Пример**:
```html
<title>Малярные работы в Минске - Покраска стен и потолков | Stroiuslugi</title>
<meta name="description" content="Малярные работы в Минске и области. Покраска стен, потолков, фасадов...">
```

### Структурированные данные
**Решение**: Использование правильной иерархии заголовков (h1, h2, h3)

**Обоснование**:
- Один h1 на странице
- Логическая иерархия заголовков
- Улучшение SEO

### Sitemap и Robots.txt
**Решение**: Создание sitemap.xml и robots.txt

**Обоснование**:
- Помощь поисковым системам в индексации
- Управление доступом роботов
- Указание приоритетов страниц

## 6. Адаптивность

### Mobile-first подход
**Решение**: Использование Bootstrap grid system с mobile-first breakpoints

**Обоснование**:
- Адаптивность из коробки через Bootstrap
- Оптимизация для мобильных устройств
- Прогрессивное улучшение для больших экранов

**Код подтверждающий**:
```html
<div class="col-md-6 col-lg-4">
```

### Адаптивная навигация
**Решение**: Collapse navbar для мобильных устройств

**Код подтверждающий**:
```html
<button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#navbarNav">
```

### Адаптивные стили
**Решение**: Медиа-запросы для мобильных устройств

**Код подтверждающий**:
```css
@media (max-width: 768px) {
    .hero h1 {
        font-size: 2rem;
    }
}
```

## 7. Оптимизация изображений

### Формат WebP
**Решение**: Использование формата WebP для изображений

**Обоснование**:
- Лучшее сжатие по сравнению с JPEG/PNG
- Сохранение качества
- Уменьшение размера файлов
- Улучшение скорости загрузки

**Паттерн именования**:
```
{category}_stroiuslugi.by_{number}.webp
```

### Fallback для изображений
**Решение**: Использование onerror для placeholder изображений

**Код подтверждающий**:
```html
<img src="img/painting/service-1.jpg" onerror="this.src='https://via.placeholder.com/400x250?text=Малярные+работы'">
```

## 8. Обработка форм

### Заглушки для backend
**Решение**: Временная обработка форм через alert

**Обоснование**:
- Прототипирование функционала
- Готовность к интеграции с backend
- TODO комментарии для будущей реализации

**Код подтверждающий**:
```javascript
// TODO: Интегрировать с backend API
alert('Спасибо за вашу заявку! Мы свяжемся с вами в ближайшее время.');
```

## 9. Навигация между страницами

### Относительные пути
**Решение**: Использование относительных путей для навигации

**Обоснование**:
- Работает на любом хостинге
- Гибкость при изменении структуры
- Поддержка локальной разработки

**Пример**:
```html
<a href="../index.html">Главная</a>
<a href="services/painting.html">Малярные работы</a>
```

## 11. Доступность

### Семантический HTML
**Решение**: Использование семантических тегов

**Примеры**:
- `<nav>` для навигации
- `<section>` для секций
- `<footer>` для футера
- `<header>` для заголовков

### ARIA атрибуты
**Решение**: Использование aria-label для улучшения доступности

**Код подтверждающий**:
```html
<nav aria-label="breadcrumb">
```

## Итоговые принципы

1. **Простота**: Минимум зависимостей, максимум нативного функционала
2. **Консистентность**: Единообразная структура всех страниц
3. **Адаптивность**: Mobile-first подход
4. **SEO-оптимизация**: Правильная структура и мета-теги
5. **Производительность**: Оптимизация изображений, минимум зависимостей
6. **Поддерживаемость**: Чистый код с комментариями и TODO
